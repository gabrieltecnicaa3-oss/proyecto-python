CAMBIOS IMPLEMENTADOS:

1️⃣  CAMPO CANTIDAD PERSONALIZADO (x de total)
────────────────────────────────────────────────
Solicitud: "Agregar campo para completar cantidad (x) manualmente y mostrar 
           'x de total' por ejemplo si hay en total 18 A1 y se envían 5 
           que muestre 5 de 18"

✓ Implementado:
  • Nueva columna en la tabla: "A Enviar"
  • Campo input numérico editable para cada pieza
  • Por defecto muestra la cantidad total
  • Se puede ingresar cantidad parcial (ej: 5 en lugar de 18)
  • Validación: No permite números > cantidad total

  Ejemplo visual en formulario web:
  ┌────────────────────────────────────────────────────────────────┐
  │ Pos. │ Total │ A Enviar │ Perfil      │ Peso  │ Descrip... │
  ├────────────────────────────────────────────────────────────────┤
  │ A1   │ 18    │ [input] ← Usuario ingresa aquí (ej: 5)          │
  │ T21  │ 1     │ [input] ← Usuario ingresa aquí                   │
  └────────────────────────────────────────────────────────────────┘

  En el PDF resultante (si el usuario ingresa 5 y 1):
  ┌────────────────────────────────────────────────────────────────┐
  │ A1  │ 18 │ 5  │ PL9.5*85   │ 2.35 │ RIGIDIZADOR...           │
  │ T21 │ 1  │ 1  │ L63.5X4.8  │ 15.92│ ARRIOSTRAMIENTO...        │
  └────────────────────────────────────────────────────────────────┘

2️⃣  REMOVER DECIMALES DEL CAMPO CANTIDAD
───────────────────────────────────────────
Solicitud: "Quitar decimales del campo cantidad"

✓ Implementado:
  • Cantidades en BD: 18.0, 1.0, etc
  • Mostradas en web: 18, 1 (sin decimales)
  • Mostradas en PDF: 18, 1 (sin decimales)

  Conversión:
    JavaScript: parseInt(parseFloat(cantidad))
    Python: int(float(cantidad))

  Ejemplo:
    BD: 18.0    → Web: 18 unidades    → PDF: 18 ✓
    BD: 1.0     → Web: 1 unidades     → PDF: 1 ✓

Columnas en orden:

  2. Posición (Ej: A1, T21)
  3. Total (Cantidad total disponible, sin decimales)
  4. A Enviar (Input numérico editable)
  5. Perfil (Ej: PL9.5*85)
  6. Peso (Ej: 2.35)
  7. Descripción (Ej: RIGIDIZADOR COL EXIST)
  8. Observaciones (TextArea para notas)

FLUJO COMPLETO DE USO

1. Usuario accede a http://127.0.0.1:5000/modulo/remito

2. Rellena el formulario:
   ├─ Orden de Trabajo: Selecciona "2 - green global / GGO-001"
   ├─ Obra: Auto-completa "GGO-001"
   ├─ Fecha de Remito: Ingresa fecha (ej: 2026-04-07)
   ├─ Transporte: Ingresa empresa/vehículo (ej: Empresa XYZ)
   └─ Se muestra la tabla con piezas

3. En la tabla ve:
   ├─ Posición A1, Total: 18, A Enviar: [input con 18 por defecto]
   └─ Posición T21, Total: 1, A Enviar: [input con 1 por defecto]

4. Usuario puede:
   ├─ Cambiar cantidad A Enviar (ej: A1 → 5, T21 → 1)
   ├─ Agregar observaciones en cada pieza
   ├─ Deseleccionar piezas si es necesario (checkbox)
   └─ Hacer clic en "📄 Generar Remito PDF"

5. Sistema genera PDF con:
   ├─ Encabezado: OT, Cliente, Obra, Fecha, Transporte
   ├─ Tabla con: POS., TOTAL, ENVIADO, PERFIL, PESO, DESCRIPCIÓN, OBS.
   └─ Datos ejemplo: A1 | 18 | 5 | PL9.5*85 | 2.35 | ... | [observaciones]

DATOS EN BD CAPTURADOS

Cuando el usuario hace POST:
  • ot_id: ID de la orden
  • fecha_remito: Fecha seleccionada
  • transporte: Campo de transporte ingresado
  • piezas[]: IDs de piezas seleccionadas
  • cant_${pieza_id}: Cantidad a enviar de cada pieza
  • obs_${pieza_id}: Observaciones de cada pieza

Ejemplo POST data:
  ot_id=2
  fecha_remito=2026-04-07
  transporte=Empresa XYZ
  piezas=174&piezas=176
  cant_174=5
  cant_176=1
  obs_174=Primera pieza
  obs_176=Segunda pieza
